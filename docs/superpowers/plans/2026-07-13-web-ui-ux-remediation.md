# Web UI UX Remediation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make `xv ui` reload-safe, resistant to stale asynchronous responses, and consistent and accessible for destructive actions.

**Architecture:** Keep the existing embedded vanilla-JavaScript UI and API unchanged. Add per-tab token recovery through `sessionStorage`, latest-operation-wins generation counters for list and drawer requests, and one reusable two-click confirmation helper for secret and file deletion.

**Tech Stack:** Rust, axum, embedded HTML/CSS/vanilla JavaScript, Cargo tests, hermetic local backend, in-app browser verification.

## Global Constraints

- Preserve the loopback-only server and existing bearer-token API authentication.
- Store the bearer token in `sessionStorage`; do not use `localStorage`, cookies, or a URL fragment.
- Do not add a JavaScript bundler, frontend framework, or production dependency.
- Stale asynchronous successes and failures must produce no visible or shared-state change.
- File deletion must use the same non-blocking two-click, three-second confirmation model as secret deletion.
- File action controls must have visible, descriptive accessible names.

- [ ] **Step 3: Add reusable confirmation state**

Implement helpers with this behavior:

```javascript
function resetConfirmation(button, label) {
  clearTimeout(button._confirmTimer);
  button._confirmTimer = null;
  button.dataset.armed = '';
  button.textContent = label;
}

function armConfirmation(button, armedLabel, timeoutMs = 3000) {
  if (button.dataset.armed === '1') return true;
  button.dataset.armed = '1';
  button.textContent = armedLabel;
  clearTimeout(button._confirmTimer);
  button._confirmTimer = setTimeout(
    () => resetConfirmation(button, button.dataset.defaultLabel),
    timeoutMs,
  );
  return false;
}
```

Set `data-default-label="Delete"` on the drawer Delete button. Call `resetConfirmation($('#delete'), 'Delete')` from both `openDrawer` and `closeDrawer`, and after current delete success/failure.
